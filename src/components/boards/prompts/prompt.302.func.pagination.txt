                - endDate: DateTime

                        search: $search
                    )
                }

            1) 테스트 제외 라이브러리
                - jest
                - @testing-library/react

            2) 테스트 조건
                - network 통신이 포함된 테스트의 timeout은 2000ms 미만으로 설정할 것.
                - network 통신이 없는 단순 연산, 렌더링 테스트는 500ms 미만 혹은 timeout 미설정으로 할 것.

            3) 테스트 API 조건
                3-1) 데이터
                - 실제 API 데이터를 사용할 것.
                - Mock데이터, MockedProvider, intercept 등 네트워크 모킹 사용 금지

                3-2) 성공 시나리오
                - 실제 API를 호출해 정상적인 응답을 수신해야 함.
                - Mock 데이터를 모킹하지 말 것.
                - 응답 데이터가 정상적으로 UI에 반영되는 지 검증할 것.

                3-3) 실패 시나리오 
                - 실제 API 호출에서 인증 실패 / 네트워크 오류 등 상황을 재현할 것.
                - Mock 데이터를 사용하거나 에러를 인위적으로 주입하지 말 것.
                - 에러 상태('error' 또는 'catch') 처리 로직이 UI에 반영되는지 검증할 것.

핵심요구사항) 다음의 조건에 맞게 페이지네이션 기능을 step-by-step으로 구현하고, 적용 결과를 체크리스트로 반환할 것.
            1) 페이지 계산 로직
                - GraphQL의 fetchBoardsCount 쿼리 응답을 기반으로 전체 게시글 수를 계산할 것.
                - rowsPerPage: 10, numPerPagegroup: 5로 설정할 것.
                - Math.ceil()을 사용하여 마지막 페이지(lastPage)를 계산할 것.
                    - 데이터가 없을 경우 기본값(rowsPerPage)로 나누어 1페이지 이상을 보장할 것.
            
            2) 현재 페이지 상태 관리
                - currentPage: 1로 설정하되, 페이지 이동 시 현재 페이지 번호를 변경할 수 있도록 할 것.
            
            3) 게시글 번호 계산
                - 현재 페이지를 기준으로 게시글의 역순 번호를 계산할 것.
                - 예) 총 게시글이 123개일 경우,
                    - 1페이지: 123~114
                    - 2페이지: 113~104
                    형태로 계산될 수 있도록 구현할 것.
                - 계산된 게시글 번호 배열은 게시글 리스트 렌더링 시 itemNumber에 번호 표시용으로 바인딩할 것.
